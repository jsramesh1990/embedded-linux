# Interrupts in Embedded Linux

## Overview

Interrupts are hardware or software signals that temporarily stop normal CPU execution to handle urgent events.

Interrupts are essential for:
- real-time responsiveness
- efficient CPU usage
- hardware communication

Without interrupts:
- CPU must continuously poll devices
- system becomes inefficient

Interrupts are one of the most important concepts in:
- Linux kernel
- embedded systems
- device drivers
- real-time systems

---

# 1. What is an Interrupt?

An interrupt is:

```text
A signal that requests immediate CPU attention
```

Generated by:
- hardware devices
- software
- timers
- exceptions

---

# 2. Why Interrupts are Needed

Without interrupts:
```text
CPU continuously checks devices (polling)
```

With interrupts:
```text
Devices notify CPU only when needed
```

---

# 3. Polling vs Interrupts

| Feature | Polling | Interrupts |
|---------|----------|------------|
| CPU Usage | High | Efficient |
| Latency | Higher | Lower |
| Complexity | Simple | More complex |
| Real-Time Response | Poor | Better |

---

# 4. High-Level Interrupt Flow

```text

         ↓

# 15. Interrupt Service Routine (ISR)

Also called:
```text
Interrupt Handler
```

Function executed when interrupt occurs.

---

# 16. ISR Characteristics

ISR should be:
- fast
- minimal
- non-blocking

Avoid:
- sleeping
- large computations

---

# 24. Top Half and Bottom Half

Linux splits interrupt handling:

| Part | Purpose |
|------|---------|
| Top Half | Immediate handling |
| Bottom Half | Deferred work |

---

# 25. Top Half

Runs immediately during interrupt.

Responsibilities:
- acknowledge device
- read critical data

---

# 26. Bottom Half

Deferred processing:
- complex operations
- networking
- filesystem work

---

# 27. Bottom Half Mechanisms

Linux supports:
- softirq
- tasklet
- workqueue

---

# 28. Interrupt Handling Split

```text
Interrupt Occurs
       ↓
Top Half (Fast)
       ↓
Schedule Bottom Half
       ↓
Return Quickly
       ↓
Bottom Half Executes Later
``` id="half1"

---

# 29. SoftIRQ
